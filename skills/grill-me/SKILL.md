---

以下の機能・要件について、設計に必要な意思決定を1問ずつ引き出します：

$ARGUMENTS

## 手順

### 1. 要件の確認

引数で渡された機能・要件を把握する。不明点があれば最初に1問だけ確認する。

### 2. 質問の実施（1問ずつ）

以下のルールで質問を行う：

- **必ず1問ずつ**質問する。複数を同時に聞かない
- 各質問は「AにしますかBにしますか？」という選択形式を基本とする
- ユーザーの回答を受けてから次の質問に進む
- 前の回答によって不要になった質問はスキップする

質問すべき領域（機能・要件に応じて適切なものを選ぶ）：
- 技術スタック・フレームワーク選定
- データ永続化方式
- 認証・認可の方式
- API設計（REST / GraphQL / RPC等）
- スケーラビリティ要件
- エラーハンドリング方針
- テスト戦略
- デプロイ環境・インフラ

### 3. 設計サマリーの出力

全質問が終わったら、決定内容を一覧にまとめて提示する：

```
## 設計サマリー: [機能名]

| 決定事項 | 選択肢 | 理由 |
|----------|--------|------|
| フレームワーク | Next.js | ... |
| 認証方式 | Magic Link | ... |
...
```
