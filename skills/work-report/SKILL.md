以下の手順で現在のセッションの作業レポートを作成します。

### 1. 環境情報の収集

まず Bash コマンドで実行環境の情報を収集します：

```bash
echo "=== 環境情報 ==="
echo "ホスト名: $(hostname)"
echo "OS: $(uname -a)"
echo "ユーザー: $(whoami)"
echo "作業ディレクトリ: $(pwd)"
echo "日時: $(date '+%Y-%m-%d %H:%M:%S %Z')"
```

必要に応じて追加情報も収集します：
- インストール済みパッケージの確認（`dpkg -l`, `pip list`, `npm list` など）
- Gitリポジトリの状態（`git log --oneline -10`, `git status`）
- サービスの状態（`systemctl status <service>`）

### 2. 会話履歴からの情報抽出

現在のセッションのやり取りを振り返り、以下の情報を整理します：

**作業概要**
- セッションの目的・背景
- 達成したこと

**インストール・設定変更**
- インストールしたパッケージ・ツール（コマンドと結果）
- 変更した設定ファイル（ファイルパスと変更内容の概要）
- 作成したファイル・ディレクトリ

**ハマった点と解決策**
- 発生したエラーや問題（エラーメッセージの要点）
- 試みた解決策
- 最終的な解決方法

**未解決の課題・次のステップ**
- 残タスク
- 今後検討すべき点

### 3. レポートの生成

以下のテンプレートに従って Markdown 形式でレポートを作成します：

```markdown
# 作業レポート - YYYY-MM-DD HH:MM

## 環境情報

| 項目 | 値 |
|------|-----|
| ホスト名 | ... |
| OS | ... |
| ユーザー | ... |
| 作業ディレクトリ | ... |

## 作業概要

（セッションの目的と達成内容を2〜5文で簡潔に）

## インストール・設定変更

### インストールしたパッケージ・ツール

- `パッケージ名` - 用途・バージョン

### 変更した設定ファイル

- `ファイルパス` - 変更内容の概要

## ハマった点と解決策

### 問題1: （タイトル）

**現象**: （エラーメッセージや症状）

**原因**: （特定できた場合）

**解決策**: （実施した対処）

## 次のステップ

- （残タスクや今後の検討事項）
```

ハマった点や解決策がなければそのセクションは省略してください。

### 4. レポートの保存

レポートを以下のファイル名で現在の作業ディレクトリに保存します：

```
${hostname}-work-report-YYYYMMDD-HHMMSS.md
```

保存後、ファイルパスをユーザーに伝えます。

ファイル名の生成例：
```bash
HOSTNAME=$(hostname)
DATE=$(date '+%Y%m%d-%H%M%S')
FILENAME="${HOSTNAME}-work-report-${DATE}.md"
```
